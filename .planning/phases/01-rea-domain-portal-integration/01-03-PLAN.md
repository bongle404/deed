    buildReaXml() implementation:
    - Use XMLBuilder with { ignoreAttributes: false, attributeNamePrefix: '@_', format: true }
    - Build the propertyList > residential structure from research code example
    - agentID: process.env.REA_AGENCY_ID
    - uniqueID: `deed-${listing.id}`
    - headline: listing.address (street only)
    - address.street: listing.address, address.suburb: listing.suburb, address.state: listing.state || 'QLD', address.postcode: listing.postcode, address.country: 'Australia'
    - price: listing.asking_price (numeric, display="yes")
    - priceView: `Offers over $${listing.asking_price.toLocaleString()}`
    - features: bedrooms, bathrooms, garages (car_spaces), landSize if listing.land_size present
    - images: map listing.photos array (if present and non-empty) to img elements with id="m", "1", "2"...
    - listingAgent: { name: 'DEED Property', email: process.env.DEED_LISTINGS_EMAIL || 'listings@deed.com.au' }

    submitViaSftp() implementation:
    - Create SftpClient, connect with REA_SFTP_HOST/REA_SFTP_USER/REA_SFTP_PASS
    - sftp.put(Buffer.from(xmlContent, 'utf8'), `/listings/${filename}`)
    - Always call sftp.end() in finally block

    handler() implementation:
    - Return 405 for non-POST
    - Return 400 if listing_id missing from req.body
    - Fetch listing from Supabase using service role
    - Return 404 if not found
    - Credential guard: const hasCredentials = !!(process.env.REA_SFTP_HOST && process.env.REA_AGENCY_ID)
    - If hasCredentials: build XML, call submitViaSftp, update rea_status='pending' and portal_submitted_at
    - If !hasCredentials: update rea_status='not_submitted', return { ok: true, mode: 'dev_skipped' }
    - On any error: update portal_error=err.message, return 500

    Do NOT use async SFTP in a way that would timeout Vercel's 10s default. The SFTP put() for a small XML file should complete in under 2 seconds on a normal connection.
