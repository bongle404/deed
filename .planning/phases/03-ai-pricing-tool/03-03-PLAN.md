  <action>
    Create `api/offer-floor.js`. Export a single async function `checkBelowFloor(offerPrice, listing)`.

    Logic:
    1. If `listing.price_estimate_low` is null or undefined, return `{ below_floor: false, below_floor_reason: null }` immediately. No Haiku call.
    2. If `offerPrice >= listing.price_estimate_low`, return `{ below_floor: false, below_floor_reason: null }`. No Haiku call.
    3. If `offerPrice < listing.price_estimate_low`:
       - Calculate percentage gap: `((listing.price_estimate_low - offerPrice) / listing.price_estimate_low * 100).toFixed(1)`
       - Format prices as AUD: `new Intl.NumberFormat('en-AU', { style: 'currency', currency: 'AUD', maximumFractionDigits: 0 }).format(price)`
       - Call Claude Haiku with this prompt:
         ```
         Write a single plain-English sentence explaining that this offer is below the comparable sales floor.
         Include all three of: the offer price ({formattedOfferPrice}), the floor price ({formattedFloor}),
         and the percentage gap ({pct}%). Do not include advice. Maximum 40 words.
         Example: "This offer of $720,000 is 8.6% below the comparable sales floor of $787,000 for similar properties in this suburb."
         ```
       - Return `{ below_floor: true, below_floor_reason: haiku_response_text }`.

    **Critical:** Claude Haiku generates the label text — the numbers (offer price, floor price, pct) are computed from real data and injected into the prompt. Haiku never invents prices.

    module.exports = { checkBelowFloor };
  </action>

    ```javascript
    const { checkBelowFloor } = require('./offer-floor');

    // After offer insert — fetch the listing's price_estimate_low
    const { data: listing } = await supabase
      .from('listings')
      .select('price_estimate_low, price_estimate_mid, price_estimate_high')
      .eq('id', offer.listing_id)  // use whatever the listing FK column is called
      .single();

    if (listing) {
      const { below_floor, below_floor_reason } = await checkBelowFloor(
        parseFloat(offer.offer_price),  // use whatever the price field is called
        listing
      );
      if (below_floor) {
        await supabase
          .from('offers')
          .update({ below_floor: true, below_floor_reason })
          .eq('id', newOfferId);  // use the actual new offer id
      }
    }
    ```

    **Important implementation notes:**

    - Only update the offer if `below_floor === true` — skip the update if false to avoid unnecessary DB writes.
    - Wrap the entire below-floor block in try/catch so it cannot break the offer flow.
